Job

Output settings

Save layout NC and reports - Enable this option for layout NC and reports to be saved.

Save layout file - Enable this option for the layout file to be saved.

Output simulation preview in reports - Enable this option to generate the tooling simulation preview along with the other outputs. This is a short GIF animation for the laser/punch tooling.

Do not delete program outputs on layout/job completion - Enabling this option will skip the delete operation for the program outputs when a job/layout is been marked as completed.

Nest settings

Mix parts from multiple Jobs - Support mixed mode nesting. Parts from multiple jobs should be nested together for better packing efficiency.

Do not auto-release layouts - This will stop layouts being available for production until all the layouts are in a released state.

Re-nesting packing cut-off - This value is used to decide whether a nested layout is held in the approval queue and is re-nested again when new parts are planned or they are released for production and related outputs are exported to the output folder.

Interactive sheet-name prefix - This section is used to support interactive nesting where the user has more control over the parts being nested and the nest result. Use a naming scheme, different from auto-nesting, for the interactively nested layouts.

Update sheet inventory on nest (un)release - If this option is enabled, the sheet inventory will update when sending a nest to a machine. Also recalling the nest from the machine will return the sheet to the inventory.

Automate remnant inventory management - When this switch is selected, the software automates the remnant inventory management. If not selected, the remnant management becomes interactive.

Auto-refresh Out-of-date layouts - When checked, the software refreshes all out-of-date layouts linked to a part after its cutting solutions are calculated.

Automate twin-line nesting - Check this option to enable twin-line automation.

Pull workflow settings

late:SH.UsePullWorkflow[Use 'Pull workflow'] - This option is useful for users who need a highly autonomous planning and nesting system. When enabled, Praxis plans the nesting on demand.

Nests per station - This value is to set how many layouts per station should get created.

Maximum sheets per nest - This value is used to control the maximum sheet count against the unique nest. So, for instance, if there is an order with a big part (which fills up the whole sheet), needed in large quantities (say 500) - Praxis no longer creates a single nest with 500 copies in pull mode. Instead, it creates 10 copies per nest where 10 is the max.

Bend programs per station - This value is to set how many bend programs per station should get created.

Mark the task completed, when the program outputs are removed - Enable this option for a layout to be marked Done when the outputs are removed from the target output destination.

Spreadsheet import settings

CAD lookup directories - Praxis will try to load the part entry mentioned in the imported CSV from the part library. However, if the part is not available in the part library then it will try to fetch the parts from look-up directories based on the order of directories set in this dialogue.

Use recursive CAD search - Enable this option for the search to be repeated for successive results.

Create a blank part for the missing CAD files - If parts do not exist in the Praxis library or lookup directories, Praxis will create a blank part for those unavailable parts.

Do not show the New Job dialog if the spreadsheet has no error - If CSV entries are clean i.e. no mismatch of data from DB or all fields are listed properly, Praxis will not show the job overview window and will process the job as soon the CSV is imported or dropped.

Prefer Raw-Material column over Material + Thickness - Supported spreadsheet types:

  1. Pattern 1: PartName, Quantity, DueDate, Priority, RawMaterial

  2. Pattern 2: PartName, Quantity, DueDate, Priority, Material, Thickness

  3. Pattern 3: PartName, Quantity, DueDate, Priority, Material, Thickness, RawMaterial

If this option is not checked, Pattern 1 will be considered the default spreadsheet type. We also support Pattern 3 in case the CSV format is of type Pattern 3.

If this option is checked and CSV is of type Pattern 2, then Praxis will resolve Raw Material from the Material and Thickness fields from CSV.

Spreadsheet output settings

Export Job/Layout completion report (.csv) - Check this option to enable CSV export when a Layout or Job is complete. The completed reports are saved at the Reports destination specified at the Factory → Settings → Reports settings.
