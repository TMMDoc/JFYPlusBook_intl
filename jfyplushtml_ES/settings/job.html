Output settings

Save layout NC and reports - Habilite esta opción para que se guarden el CN de diseño y los informes.

Save layout file - Habilite esta opción para que se guarde el archivo de diseño.

Output simulation preview in reports - Habilite esta opción para generar la vista previa de simulación de utillaje junto con las otras salidas. Esta es una breve animación GIF para el utillaje láser/de punzonado.

Do not delete program outputs on layout/job completion - Al habilitar esta opción, se omitirá la operación de eliminación de las salidas del programa cuando se marque un trabajo/diseño como completado.

Nest settings

Mix parts from multiple Jobs - Admite nesting en modo mixto. Las piezas de varias tareas deben anidarse juntas para una mejor eficiencia de empaquetado.

Do not auto-release layouts - Esto evitará que los diseños estén disponibles para la producción hasta que todos los diseños estén en un estado liberado.

Re-nesting packing cut-off - Este valor se utiliza para decidir si un diseño anidado se mantiene en la cola de aprobación y se vuelve a anidar cuando se planifican nuevas piezas o se liberan para producción y las salidas relacionadas se exportan a la carpeta de salida.

Interactive sheet-name prefix - Esta sección se utiliza para admitir el nesting interactivo, donde el usuario tiene más control sobre las partes que se anidan y el resultado del nesting. Utilice un esquema de nomenclatura, diferente del nesting automático, para los diseños anidados de forma interactiva.

Update sheet inventory on nest (un)release - Si esta opción está habilitada, el inventario de chapas se actualizará al enviar un nesting a una máquina. Además, retirar el nesting de la máquina devolverá la chapa al inventario.

Automate remnant inventory management - Cuando se selecciona este interruptor, el software automatiza la gestión del inventario remanente. Si no se selecciona, la gestión de remanentes se vuelve interactiva.

Auto-refresh Out-of-date layouts - Cuando se marca, el software actualiza todos los diseños obsoletos vinculados a una pieza después de calcular sus soluciones de corte.

Automate twin-line nesting - Marque esta opción para habilitar la automatización de doble línea.

Pull workflow settings

late:SH.UsePullWorkflow[Use 'Pull workflow'] - Esta opción es útil para los usuarios que necesitan un sistema de planificación y nesting altamente autónomo. Cuando está habilitado, Praxis planifica el nesting bajo demanda.

Nests per station - Este valor es para establecer cuántos diseños por estación se deben crear.

Maximum sheets per nest - Este valor se utiliza para controlar el recuento máximo de chapas frente al nesting único. Así, por ejemplo, si hay un pedido con una gran pieza (que llena toda la chapa), necesaria en grandes cantidades (digamos 500), Praxis ya no crea un solo nesting con 500 copias en modo pull. En su lugar, crea 10 copias por nesting, donde 10 es el máximo.

Bend programs per station - Este valor es para establecer cuántos programas de plegado por estación deben crearse.

Mark the task completed, when the program outputs are removed - Habilite esta opción para que se marque un diseño como Hecho cuando se eliminen las salidas del destino de salida objetivo.

Spreadsheet import settings

CAD lookup directories - Praxis intentará cargar la entrada de pieza mencionada en el CSV importado desde la biblioteca de piezas. Sin embargo, si la pieza no está disponible en la biblioteca de piezas, intentará obtener las piezas de los directorios de búsqueda en función del orden de los directorios establecidos en este diálogo.

Use recursive CAD search - Habilite esta opción para que la búsqueda se repita para resultados sucesivos.

Create a blank part for the missing CAD files - Si las piezas no existen en la biblioteca de Praxis o en los directorios de búsqueda, Praxis creará una pieza bruta para aquellas piezas no disponibles.

Do not show the New Job dialog if the spreadsheet has no error - Si las entradas CSV están limpias, es decir, no hay falta de coincidencia de los datos de la base de datos o todos los campos se enumeran correctamente, Praxis no mostrará la ventana de descripción general de la tarea y procesará la tarea tan pronto como se importe o se elimine el CSV.

Prefer Raw-Material column over Material + Thickness - Tipos de hojas de cálculo compatibles:

  1. Patrón 1: Nombre de pieza, Cantidad, Fecha de vencimiento, Prioridad, Materia prima

  2. Patrón 2: Nombre de pieza, Cantidad, Fecha de vencimiento, Prioridad, Material, Espesor

  3. Patrón 3: Nombre de pieza, Cantidad, Fecha de vencimiento, Prioridad, Material, Espesor, Materia prima

Si esta opción no está marcada, el patrón 1 se considerará el tipo de hoja de cálculo predeterminado. También admitimos el Patrón 3 en caso de que el formato CSV sea del tipo Patrón 3.

Si esta opción está marcada y el CSV es del tipo Patrón 2, Praxis resolverá la Materia prima de los campos Material y Espesor del CSV.

Spreadsheet output settings

Export Job/Layout completion report (.csv) - Marque esta opción para habilitar la exportación CSV cuando se complete un diseño o una tarea. Los informes completados se guardan en el destino de informes especificado en Fábrica → Ajustes → Ajustes de informes.
