Benachrichtigungen

Notification settings

Enable email notifications – Mit dieser Option wird die E-Mail-Benachrichtigung aktiviert und konfiguriert.

Notification recipients – Geben Sie die Empfängerdetails ein. Um mehr als einen Empfänger einzugeben, verwenden Sie ein Leerzeichen oder Komma, um sie zu trennen.

Send messages in batch – Diese Option wird verwendet, um mehrere Nachrichten zu kombinieren und als Massenaktion zu senden. Um große Nachrichten zu vermeiden, werden Nachrichten, die innerhalb eines bestimmten Zeitraums gesendet werden, kombiniert.

Attach tooling simulation preview where applicable (Warning: Messages may become too large) – Diese Option fügt der E-Mail-Benachrichtigung eine Werkzeugsimulationsvorschau hinzu.

Mail settings

SMTP server – Die SMTP-E-Mail-Server-Adresse finden Sie im Abschnitt „Konto“ oder „Einstellungen“ des E-Mail-Clients des Benutzers.

SMTP port – Diese Einstellung ist der grundlegende Standard, den Mailserver verwenden, um E-Mails über das Internet zu versenden.

Transport protocol –

Server needs authentication – Aktivieren Sie diese Option, damit der Server validiert wird.

User-name – Geben Sie hier die E-Mail-Adresse des Empfängers ein.

Password – Geben Sie hier das Passwort des Empfängers ein.

Test…​ – Verwenden Sie den Testschalter, um zu überprüfen, ob eine E-Mail erfolgreich an den Benutzer zugestellt wurde. Eine Fehlermeldung wird wie im Bild unten angezeigt, wenn die Test-E-Mail fehlgeschlagen ist.

Die E-Mail-Benachrichtigung funktioniert nicht im interaktiven Modus. Wenn Sie also ein Teil bearbeiten, eine Änderung anwenden und speichern, wird keine Benachrichtigung an den Benutzer gesendet, selbst wenn ein Rüst-/Konstruktionsfehler in dem zu speichernden Teil vorliegt.
Aufgrund der Art der E-Mail-Kommunikation kann die Zustellung viel langsamer sein und wird daher im Hintergrund ausgeführt. Benachrichtigungen können verloren gehen, wenn die Engine gestoppt/heruntergefahren wird, bevor alle Nachrichten zugestellt sind.
