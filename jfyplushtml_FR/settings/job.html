Tâche

Output settings

Save layout NC and reports - Activez cette option pour enregistrer la CN Layout et les rapports.

Save layout file - Activez cette option pour enregistrer le fichier de layout.

Output simulation preview in reports - Activez cette option pour générer l’aperçu de la simulation de montage avec les autres sorties. Il s’agit d’une courte animation GIF pour le montage de laser/poinçon.

Do not delete program outputs on layout/job completion - Lorsque cette option est activée, l’opération de suppression est ignorée pour les sorties du programme si une tâche ou un layout est marqué comme terminé.

Nest settings

Mix parts from multiple Jobs - Prend en charge l’imbrication en mode mixte. Les pièces de plusieurs tâches doivent être imbriquées ensemble pour une distribution plus efficace.

Do not auto-release layouts - Empêche la mise à disposition des layouts pour la production jusqu’à ce que tous les layouts soient à l’état Libéré.

Re-nesting packing cut-off - Cette valeur est utilisée pour décider si un layout imbriqué est conservé dans la file d’attente d’approbation et est réimbriqué lorsque de nouvelles pièces sont planifiées ou lorsqu’elles sont libérées pour la production et que les sorties associées sont exportées vers le dossier de sortie.

Interactive sheet-name prefix - Cette section est utilisée pour prendre en charge l’imbrication interactive pour laquelle l’utilisateur a plus de contrôle sur les pièces imbriquées et le résultat de l’imbrication. Utilisez un schéma de dénomination, différent de l’imbrication automatique, pour les layouts imbriqués de manière interactive.

Update sheet inventory on nest (un)release - Si cette option est activée, le stock magasin de tôles sera mis à jour après l’envoi d’une imbrication à une machine. Lorsque vous rappeler une imbrication depuis une machine, la tôle sera renvoyée dans le stock.

Automate remnant inventory management - Lorsque ce commutateur est sélectionné, le logiciel automatise la gestion des stocks restants. S’il n’est pas sélectionné, la gestion des restes devient interactive.

Auto-refresh Out-of-date layouts - Lorsque cette option est cochée, le logiciel actualise tous les layouts obsolètes liés à une pièce après le calcul du programme de coupe.

Automate twin-line nesting - Cochez cette option pour activer l’automatisation TwinLine.

Pull workflow settings

late:SH.UsePullWorkflow[Utiliser 'Pull workflow'] - Cette option est utile pour les utilisateurs qui ont besoin d’un système de planification et d’imbrication hautement autonome. Lorsqu’elle est activée, Praxis planifie l’imbrication à la demande.

Nests per station - Cette valeur permet de définir le nombre de layout à créer par station.

Maximum sheets per nest - Cette valeur est utilisée pour contrôler le nombre maximal de tôle par rapport à l’imbrication unique. Par exemple, si un ordre comprend beaucoup de pièces (ce qui remplit toute la tôle) et doit être produit en grande quantité (disons 500) - Praxis ne crée plus une imbrication unique avec 500 exemplaires en mode pull. Au lieu de cela, il crée 10 copies par imbrication, pour laquelle 10 est le maximum.

Bend programs per station - Cette valeur permet de définir le nombre de programmes de pliage à créer par station.

Mark the task completed, when the program outputs are removed - Activez cette option pour qu’un layout soit marqué comme terminé lorsque les sorties sont supprimées de la destination de sortie cible.

Spreadsheet import settings

CAD lookup directories - Praxis essaie de charger l’entrée de pièce mentionnée dans le fichier CSV importé à partir de la bibliothèque de pièces. Cependant, si la pièce n’est pas disponible dans la bibliothèque de pièces, Praxis essaie d’extraire les pièces des répertoires de recherche en fonction de l’ordre des répertoires défini dans cette fenêtre de dialogue.

Use recursive CAD search - Activez cette option pour que la recherche soit répétée pour les résultats successifs.

Create a blank part for the missing CAD files - Si des pièces n’existent pas dans la bibliothèque Praxis ou dans les répertoires de recherche, Praxis crée une pièce vierge pour ces pièces indisponibles.

Do not show the New Job dialog if the spreadsheet has no error - Si les entrées CSV sont propres, c’est-à-dire qu’il n’y a pas de problème de concordance entre les données de la base de données ou que tous les champs sont répertoriés correctement, Praxis n’affiche pas la fenêtre Aperçu des tâches et traite la tâche dès que le CSV est importé ou supprimé.

Prefer Raw-Material column over Material + Thickness - Types de feuilles de calcul pris en charge :

  1. Modèle 1 : Nom de la pièce, Quantité, Date d’échéance, Priorité, Matière brute

  2. Motif 2 : Nom de la pièce, Quantité, Date d’échéance, Priorité, Matière, Epaisseur

  3. Modèle 3 : Nom de la pièce, Quantité, Date d’échéance, Priorité, Matière, Epaisseur, Matière première

Si cette option n’est pas cochée, le modèle 1 est considéré comme le type de feuille de calcul par défaut. Nous prenons également en charge le modèle 3 dans le cas où le format CSV est de type modèle 3.

Si cette option est cochée et que le CSV est de type modèle 2, Praxis résoudra les matières premières à partir des champs Material et Epaisseur du CSV.

Spreadsheet output settings

Export Job/Layout completion report (.csv) - Cochez cette option pour activer l’exportation CSV lorsqu’un layout ou une tâche est terminé. Les rapports terminés sont enregistrés à la destination Rapports spécifiée dans Usine → Réglages → Réglages de rapports.
